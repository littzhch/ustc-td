# 《科大守卫战》美术素材制作文档 v1.0

## 1. 文档目的

本文档用于指导《科大守卫战》第一迭代的网页端游戏素材制作。目标不是做一张“只能看不能做”的概念图，而是产出可以直接接入 Phaser / Canvas + HTML/CSS 的素材包。

本项目美术方向为：**清新、现代、卡通校园风、轻量塔防、2.5D 俯视视角**。所有素材应保持统一的线条粗细、光影角度、饱和度和圆润程度，避免写实、厚涂、暗黑、赛博朋克、复杂透视。

---

## 2. 实现原则

### 2.1 渲染分层

| 层级 | 内容 | 推荐实现方式 |
|---|---|---|
| 背景层 | 草地、道路、河流、建筑、树木、喷泉、篮球场 | 一张地图底图 PNG，后期可拆成 Tilemap |
| 玩法层 | 塔位、防御塔、敌人、子弹、血条、范围圈 | Phaser Sprite / Graphics |
| UI 层 | 顶部状态栏、右侧详情、建造栏、强化弹窗 | HTML/CSS |

### 2.2 素材制作原则

1. 所有动态对象必须能从背景中清晰分离。
2. 防御塔、敌人、塔位应优先保证识别度，而不是追求复杂细节。
3. 所有 PNG 素材建议使用透明背景。
4. 同一类素材保持相同画布尺寸，方便写 Sprite 配置。
5. 所有图标避免包含文字，文字由 HTML/CSS 渲染。
6. AI 生成素材后必须二次裁剪、去底、压缩和命名。

## 3. 美术风格规范

### 3.1 整体风格

- 关键词：现代大学、清新校园、卡通塔防、轻策略、明亮、干净、圆润、2.5D。
- 视角：轻微俯视 3/4 视角，接近网页塔防常见视角。
- 色彩：草地浅绿、建筑白蓝、UI 白底蓝色强调、金币金黄、危险红色。
- 光影：统一左上方受光，右下方柔和阴影。
- 线条：中等粗细深色描边，不要太黑，建议深蓝灰描边。

### 3.2 色彩建议

| 用途 | 色值建议 | 说明 |
|---|---|---|
| 主蓝色 | #1E7CF2 | 按钮、校徽、科技感高亮 |

| 辅助灰 | #6B778C | 次级文字 |
| 草地绿 | #8DD66B / #AEE37F | 地图主色 |
| 道路米白 | #F1E8D8 | 主路径 |
| 金币黄 | #F7B731 | 资源图标 |
| 生命红 | #E74C3C | 血量和警示 |

## 6. 切图与导出规范

### 6.1 文件格式

| 类型 | 格式 | 背景 | 推荐尺寸 |
|---|---|---|---|
| 地图底图 | PNG / WebP | 不透明 | 1040×960 |
| 防御塔 | PNG / WebP | 透明 | 256×256 |
| 敌人 | PNG / WebP | 透明 | 160×160 或 180×180 |
| 图标 | PNG / SVG | 透明 | 64×64 |
| 特效 | PNG / WebP | 透明 | 64×64 / 128×128 |

### 6.2 命名规范

文件名统一使用小写英文和下划线：

```text
map_east_campus_01.png
tower_math.png
tower_physics.png
tower_lab.png
tower_coffee.png
enemy_homework.png
enemy_report.png
enemy_ddl.png
icon_hp.png
icon_coin.png
fx_bullet_blue.png
```

### 6.3 透明边距

- 256×256 防御塔：主体占 180–220px，四周保留 16–32px 透明边距。
- 160×160 敌人：主体占 110–140px，底部可保留少量阴影。
- 图标：主体占画布 70% 左右，不要贴边。

## 9. 质量检查清单

交付素材前逐项检查：

- [ ] 所有动态素材都是透明背景。
- [ ] 防御塔和敌人缩小到 64×64 后仍能识别。
- [ ] 同一类素材视角统一，没有明显透视冲突。
- [ ] 所有素材风格统一，不混用写实、像素、厚涂、扁平等风格。
- [ ] 地图道路清晰，敌人路径 3 秒内可辨认。
- [ ] 塔位足够明显，但不抢画面主体。
- [ ] 素材文件名符合命名规范。
- [ ] 文件已压缩，网页加载不会明显卡顿。
- [ ] UI 图标不含文字，文字全部交给 HTML/CSS。
- [ ] 地图中涉及校园元素时，不出现错误、误导或争议内容。
